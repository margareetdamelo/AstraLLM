# Astra LLM 业务分析文档

## 1 项目概述

### 1.1 项目定位

Astra LLM 是一个专为 **AsterDEX** 去中心化交易所设计的**机构级AI量化交易平台**。该平台将先进的人工智能模型与 AsterDEX 基础设施深度集成，实现自动化交易策略的执行。

**核心特性：**
- 专为 AsterDEX 优化的 AI 交易系统
- 实时市场监控与分析
- 多策略动态切换
- 自动化订单执行
- 全面的风险管理

### 1.2 目标市场

- **交易所**: AsterDEX (AsterDex Network)
- **交易类型**: 合约期货 (Futures)
- **网络**: Binance Smart Chain (BSC)

## 4 状态机设计

### 4.1 机器人状态机

```mermaid
stateDiagram-v2
    [*] --> 初始化中
    
    初始化中 --> 空闲: 初始化完成
    
    空闲 --> 运行中: start() 调用
    运行中 --> 暂停: stop() 调用
    暂停 --> 运行中: start() 调用
    运行中 --> 空闲: 停止交易
    
    状态说明:
    - 初始化中: 加载配置、连接API、初始化策略
    - 空闲: 机器人已启动但未开始交易循环
    - 运行中: 执行交易循环 (每60秒迭代)
    - 暂停: 交易循环停止但系统保持运行
```

### 4.2 市场状态机

```mermaid
stateDiagram-v2
    [*] --> UNKNOWN
    
    UNKNOWN --> HIGH_VOLATILITY_TRENDING: 波动率>3% + ADX>40 + 动量强
    UNKNOWN --> LOW_VOLATILITY_RANGING: 波动率<1.5% + ADX<25 + 动量弱
    UNKNOWN --> MOMENTUM_EXHAUSTION: RSI极端(>75或<25)
    
    HIGH_VOLATILITY_TRENDING --> MOMENTUM_EXHAUSTION: 动量减弱 + RSI极端
    HIGH_VOLATILITY_TRENDING --> LOW_VOLATILITY_RANGING: 波动率降低
    HIGH_VOLATILITY_TRATILITY_TRENDINGENDING --> HIGH_VOL: 趋势延续
    
    LOW_VOLATILITY_RANGING --> HIGH_VOLATILITY_TRENDING: 突破震荡区间
    LOW_VOLATILITY_RANGING --> MOMENTUM_EXHAUSTION: 极端资金费率
    LOW_VOLATILITY_RANGING --> LOW_VOLATILITY_RANGING: 区间震荡延续
    
    MOMENTUM_EXHAUSTION --> HIGH_VOLATILITY_TRENDING: 反转后趋势形成
    MOMENTUM_EXHAUSTION --> LOW_VOLATILITY_RANGING: 反转后回归区间
    MOMENTUM_EXHAUSTION --> MOMENTUM_EXHAUSTION: 极端状态持续
    
    状态特征:
    - HIGH_VOLATILITY_TRENDING: 趋势明显、波动大
    - LOW_VOLATILITY_RANGING: 区间震荡、波动小
    - MOMENTUM_EXHAUSTION: 超买超卖、可能反转
    - MIXED: 状态不明确
    - UNKNOWN: 初始状态
```

### 4.3 持仓状态机

```mermaid
stateDiagram-v2
    [*] --> 无持仓
    
    无持仓 --> 开仓中: 策略信号 + 风险检查通过
    开仓中 --> 持仓中: 订单成交
    开仓中 --> 开仓失败: 订单失败
    
    持仓中 --> 止盈平仓: 触发止盈条件
    持仓中 --> 止损平仓: 触发止损条件
    持仓中 --> 手动平仓: 用户手动平仓
    持仓中 --> 持仓中: 继续持仓
    
    开仓失败 --> 无持仓: 清理订单
    止盈平仓 --> 无持仓: 记录盈利交易
    止损平仓 --> 无持仓: 记录亏损交易
    手动平仓 --> 无持仓: 记录交易
    
    重要事件:
    - 持仓中会定期同步交易所持仓状态
    - 平仓后自动取消关联的止损/止盈订单
    - 所有交易记录保存到SQLite数据库
```

## 6 风险管理体系

### 6.1 风险参数

| 参数 | 默认值 | 说明 |
|-----|-------|------|
| `max_leverage` | 5x | 最大杠杆倍数 |
| `risk_per_trade` | 0.5% | 单笔风险占比 |
| `max_daily_loss` | 2% | 日最大亏损限制 |
| `max_open_positions` | 2 | 最大同时持仓数 |

### 6.2 风险检查流程

```
开仓前检查 (can_open_position):
    │
    ├─▶ 日亏损限制检查
    │      条件: daily_pnl >= -(initial_capital × 2%)
    │      不通过 → 拒绝开仓
    │
    ├─▶ 最大持仓数检查
    │      条件: current_positions < 2
    │      不通过 → 拒绝开仓
    │
    ├─▶ 重复交易对检查
    │      条件: symbol not in current_positions
    │      不通过 → 拒绝开仓
    │
    └─▶ 资金充足检查
           条件: current_capital > initial_capital × 50%
           不通过 → 拒绝开仓
```

### 6.3 仓位计算

```python
def calculate_position_size(entry_price, stop_loss, leverage, volatility):
    # 1. 计算风险金额
    risk_amount = current_capital × 0.5%
    
    # 2. 波动率调整 (高波动时减少仓位)
    if volatility:
        volatility_multiplier = max(0.5, 1 - (volatility × 2))
        risk_amount *= volatility_multiplier
    
    # 3. 计算仓位
    stop_distance = abs(entry_price - stop_loss) / entry_price
    position_value = risk_amount / stop_distance
    quantity = position_value / entry_price
    
    # 4. 杠杆调整 (高杠杆减少仓位)
    leverage_adjustment = min(1.0, 20 / leverage)
    quantity *= leverage_adjustment
    
    return quantity
```

## 7 策略选择逻辑

### 7.1 市场状态与策略匹配

| 市场状态 | 推荐策略 (优先级排序) |
|---------|---------------------|
| HIGH_VOLATILITY_TRENDING | 1. Breakout Scalping<br>2. Momentum Reversal<br>3. Support/Resistance Bounce<br>4. Order Flow Imbalance |
| LOW_VOLATILITY_RANGING | 1. Market Making<br>2. VWAP Reversion<br>3. Order Flow Imbalance<br>4. Support/Resistance Bounce |
| MOMENTUM_EXHAUSTION | 1. Momentum Reversal<br>2. VWAP Reversion<br>3. Support/Resistance Bounce<br>4. Order Flow Imbalance |
| MIXED | 1. Order Flow Imbalance<br>2. VWAP Reversion<br>3. Breakout Scalping<br>4. Market Making |

### 7.2 策略评分算法

```
final_score = (regime_score × 0.7) + (performance_score × 0.3)

其中:
 策略与当前市场状态的匹配程度- regime_score: (0-1)
- performance_score: 策略历史表现 (0-1)
    = (近期胜率 × 0.6) + (近期平均盈亏 × 0.4, 封顶0.4)

自动禁用: 胜率 < 35% 且交易数 ≥ 10 的策略
自动启用: 胜率 > 45% 的策略
```

## 10 总结

### 10.1 核心价值

1. **自动化**: 全自动交易流程，无需人工干预
2. **智能化**: 8种AI策略，基于市场状态动态选择
3. **安全化**: 多层风险控制，保护资金安全
4. **透明化**: 完整的数据记录和统计
5. **可扩展**: 灵活的策略架构，便于添加新策略

### 10.2 业务流程总结

```
启动 → 初始化 → 循环迭代(每60秒):
  1. 获取市场数据(K线/订单簿/资金费率)
  2. 检测市场状态
  3. 选择最优策略
  4. 生成交易信号
  5. 风险检查
  6. 执行订单
  7. 监控持仓
  8. 记录交易
```

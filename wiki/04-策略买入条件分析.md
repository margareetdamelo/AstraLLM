## 二、详细条件说明

### 2.1 FILTER #1: ATR 波动率检查

**目的**: 确认市场处于盘整期，波动收窄

**计算公式**:
```
ATR = True Range的14日均值
ATR百分比 = (ATR / 当前价格) × 100
```

**条件**:
```
ATR百分比 < 1.5%
```

**原理**: 
- 低ATR表示价格波动收窄，市场在积蓄能量
- 高ATR表示市场波动剧烈，不适合突破交易

**示例**:
- BTC价格 $42,000，ATR = $420 → ATR% = 1% ✓ 通过
- BTC价格 $42,000，ATR = $840 → ATR% = 2% ✗ 不通过

---

### 2.2 FILTER #2: 成交量倍数检查

**目的**: 确认是真实突破而非假突破

**计算公式**:
```
成交量倍数 = 当前成交量 / 20日平均成交量
```

**条件**:
```
成交量倍数 ≥ 4.0x
```

**原理**:
- 真实突破通常伴随巨量成交
- 成交量4x以上表示有大资金入场
- 低成交量突破往往是假突破

**置信度加分**:
| 成交量倍数 | 置信度加成 |
|------------|-----------|
| ≥ 5.0x | +20% |
| ≥ 4.0x | +15% |
| ≥ 3.0x | +12% |
| < 3.0x | +5% |

---

### 2.3 FILTER #3: RSI 健康检查

**目的**: 避免在极端位置入场

**条件**:
```
35 < RSI < 65
```

**原理**:
- RSI < 30: 超卖，可能反弹
- RSI > 70: 超买，可能回调
- RSI 40-60: 健康区间，主趋势延续概率高

**置信度加分**:
| RSI范围 | 置信度加成 |
|---------|-----------|
| 40-60 (距离50<10) | +15% |
| 35-65 (距离50<20) | +10% |
| 其他范围 | +5% |

---

### 2.4 FILTER #4: 假突破历史检查

**目的**: 避免在频繁假突破的区域交易

**检查方法**:
- 检查过去30根K线（排除最近10根）
- 统计突破后又反转的情况
- 超过2次假突破则不交易

**条件**:
```
假突破次数 ≤ 2
```

---

### 2.5 FILTER #5: 突破检测

**目的**: 确认价格突破盘整区间

**计算方法**:
```
最近30根K线高点 = MAX(high[-30:])
最近30根K线低点 = MIN(low[-30:])

向上突破: 价格 > 最近30根高点
向下突破: 价格 < 最近30根低点
```

**突破强度计算**:
```
向上突破强度 = (价格 - 近期高点) / 近期高点
向下突破强度 = (近期低点 - 价格) / 近期低点
```

## 三、置信度计算

### 3.1 置信度组成

置信度基础值为 **50%**，根据以下条件加分：

| 条件 | 加分 |
|------|------|
| 成交量 ≥ 5.0x | +20% |
| 成交量 ≥ 4.0x | +15% |
| 成交量 ≥ 3.0x | +12% |
| ATR < 1.0% | +15% |
| ATR < 1.5% | +12% |
| ATR < 2.0% | +8% |
| RSI距离50 < 10 | +15% |
| RSI距离50 < 20 | +10% |
| 突破强度 > 1.5% | +10% |
| 突破强度 > 1.0% | +8% |
| 无假突破历史 | +10% |

**最终置信度**: 基础50% + 各条件加分，最高100%

### 3.2 最小置信度要求

```
最小置信度 = 75%
```

只有置信度 ≥ 75% 时才会执行买入

## 六、为什么6个月回测只有1笔交易？

### 6.1 过滤器过于严格

| 过滤器 | 阈值 | 1小时K线满足概率 |
|--------|------|-----------------|
| ATR < 1.5% | 极严格 | ~5-10% |
| 成交量 ≥ 4x | 非常严格 | ~1-5% |
| RSI 35-65 | 严格 | ~50% |
| 假突破 ≤ 2次 | 中等 | ~60% |

**综合概率**: 约 5% × 5% × 50% × 60% = **0.075%**

即平均每1333根K线（约55天）才可能产生1个信号

### 6.2 结论

该策略设计为**极高置信度**交易系统，目标是75%+胜率，因此会错过大量机会。这是设计如此，不是bug。
